Partner Unique Email

Enforce unique, normalized emails across contacts and companies

This module prevents duplicate partner emails by normalizing (trim + lowercase) and enforcing uniqueness on a stored field. Empty emails are allowed; duplicates are blocked regardless of case or whitespace.

Key Features

  • Normalization of emails on create/write
  • SQL constraint + Python constraint for safety
  • Case-insensitive duplicate detection
  • Works for companies and contacts
  • Allows multiple empty emails (no false positives)
  • No extra configuration required
  • Compatible with Odoo 17

How it works

On every create/write, the module stores a trimmed, lowercase version of the email into email_normalized. A uniqueness constraint on that field blocks duplicates. A Python constraint adds an explicit validation message.
